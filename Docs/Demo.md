# VR GAME

## 最终方向：VR 音游跑酷 Demo

我们当前决定制作一个结合 VR 双手输入、节奏判定和跑酷演出的音乐游戏 Demo。玩家不需要真实奔跑，也不需要看到自己的完整角色建模；核心体验是：玩家在音乐节拍上完成左右手按键、手部朝向和持续按压等输入，游戏中的跑酷者根据这些输入自动完成一段线性跑酷流程，最终抵达终点传送门。

### Demo 目标

- 做一个短流程、线性、可演示的 VR 音游跑酷关卡。
- 只实现 5 类基础跑酷动作，每类动作对应一种明确的玩家输入。
- 每个动作暂时只绑定一段固定动画和一组固定环境资产。
- 不制作玩家完整角色建模，重点表现第一人称视角、手部输入反馈、环境运动和跑酷结构。
- 通过音乐节奏编排，把 5 种动作组合成一段完整流程。
- 目标体验是“用双手和节奏驱动跑酷”，而不是自由移动或真实物理攀爬。

### 核心玩法循环

1. 音乐开始，玩家进入开放宇宙背景中的悬空跑酷场景。
2. 节拍提示按照音乐节奏出现，提示玩家下一步需要完成的手部输入。
3. 玩家在判定窗口内完成对应输入。
4. 输入成功时，角色完成对应跑酷动作，场景继续向前推进。
5. 输入失败或漏键时，角色仍然会执行对应动作，保证跑酷演出不被立刻打断，但会触发失败音效、画面震荡和节奏惩罚。
6. 每次失败会降低音乐播放速度和整体节奏速度，例如失败一次后音乐速度变为 0.97 倍。
7. 音乐速度变化会同步影响 BPM、输入判定窗口、动作触发节奏和关卡推进节奏。
8. 如果玩家持续失败，速度低于 0.9 后，音乐开始发生畸变，画面和空间也进入失稳状态。
9. 如果速度继续低于最终失败阈值，玩家会被身后的红色危险区域或空间异常吞噬，关卡失败。
10. 玩家连续完成动作组合并维持节奏稳定，最终抵达终点处的传送门。

### 五种基础动作

#### 1. 交替踏步 / 跨越空隙

- 输入方式：用户交替按下左手按键和右手按键。
- 对应节奏：固定节拍，适合作为关卡的基础行进节奏。
- 游戏表现：角色在一组间隔排列的悬空地板之间前进，类似连续跨过地面空隙。
- 环境资产：长条形或矩形的悬浮地板，按照节奏等距排列。
- 设计作用：建立基础节奏感，让玩家快速理解“按键 = 跑酷步伐”。

#### 2. 侧向抓手 / 悬空横移

- 输入方式：指定某只手朝向左边或右边，并按下对应手的按键。
- 对应节奏：可以放在普通踏步之后，制造左右方向变化。
- 游戏表现：脚下悬空，角色借助左侧或右侧的悬浮抓手结构向前移动。
- 环境资产：漂浮在路线两侧的三角形抓点、发光抓手、斜切凸起或短墙片段。
- 设计作用：引入手部方向判断，让玩家不只是按键，还要把手摆到正确方位。

#### 3. 双手下压 / 滑铲

- 输入方式：两只手同时朝下，并同时按下按键。
- 对应节奏：适合放在障碍物即将擦过头顶的重拍位置。
- 游戏表现：角色压低身体完成滑铲，从低矮障碍或压迫结构下方穿过。
- 环境资产：低矮横梁、下压式门框、发光障碍带、贴近头顶的几何结构。
- 设计作用：制造身体姿态变化，让玩家感到自己参与了动作幅度更大的跑酷。

#### 4. 双手上举 / 长距离跳跃

- 输入方式：两只手同时朝上，并同时按下按键。
- 对应节奏：适合用在段落高潮或强拍上。
- 游戏表现：角色进行长距离跳跃，跨越大空隙或从一个平台跃向另一个平台。
- 环境资产：远距离平台、上方牵引光带、发光跳跃门、空中轨迹线。
- 设计作用：制造关卡中的视觉高潮，让玩家通过双手上举获得“起跳”的参与感。

#### 5. 单手上举持续按压 / 钩锁前进

- 输入方式：指定某只手朝上，并持续按下按键。
- 对应节奏：适合做成一小段持续音、长音或音乐过渡段。
- 游戏表现：角色发射或抓住钩锁，沿弧线或直线向前滑行。
- 环境资产：上方钩锁锚点、发光牵引线、环形挂点、空中轨道。
- 设计作用：引入持续输入，让关卡节奏不只是离散按键，也有拉长动作和速度变化。

### 输入判定原则

- 每个动作只对应一个明确输入，降低 Demo 复杂度。
- 判定内容包括：左右手、是否按键、手部方向、是否同时、是否持续。
- 手部方向可以先用粗略区域判断，例如上、下、左、右四个方向。
- 节奏判定可以使用固定时间窗口，例如 Perfect、Good、Miss。
- Demo 阶段不追求复杂评分系统，只需要判断动作成功与失败。
- 成功反馈要明显：音效、环境光闪烁、速度推进、动作动画衔接。
- 失败反馈也要清晰：失败音效、画面震荡、红色增强、音乐降速、BPM 失稳和空间压迫增强。
- 输入失败时动作仍然播放，不立刻中断跑酷流程；惩罚重点放在音乐速度、节奏压力和世界状态变化上。

### 动态 BPM / 失稳失败系统

- 游戏失败不是一次漏键就立刻终止，而是通过音乐速度和世界状态逐步恶化。
- 初始速度倍率为 1.0。
- 每次输入失败或漏键时，角色仍然执行动作，但触发失败音效、画面震荡和速度惩罚。
- 速度惩罚示例：失败一次后音乐速度变为 0.97 倍，后续失败继续降低。
- 当前速度倍率会统一作用于音乐播放、BPM、Beat Event 时间轴、输入窗口、动作动画节奏和关卡推进速度。
- 当速度倍率低于 0.9 时，进入警戒状态：音乐开始轻微畸变，背景和光效出现不稳定脉冲。
- 当速度倍率低于最终失败阈值时，进入失败状态：身后红色危险区域逼近，玩家被吞噬，关卡结束。
- 这个系统可以作为 Demo 的核心特色，让玩家感到自己不是单纯扣分，而是在把整首音乐和整个空间拖入失控。
- 连续成功会逐渐恢复速度倍率，最多恢复到正常倍率 1.0。
- 恢复速度应小于失败惩罚，形成“失败会快速失稳，稳定演奏可以慢慢救回来”的压力曲线。
- 示例规则：失败一次速度倍率降低到 0.97；连续成功若干次后，每次小幅恢复，例如 +0.01，直到回到 1.0。
- 这个恢复机制让玩家在失误后仍有挽回空间，同时强化“把音乐重新拉回正轨”的体验。

### 关卡结构

- 整体是线性流程，玩家从起点沿开放空间中的悬浮轨道向前跑酷，到达终点传送门。
- 场景结构悬空，脚下不是连续地面，而是由不同跑酷模块拼接出来的路径。
- 背景不做封闭隧道，而是开放宇宙空间：远处有星空、抽象星云、网格地平线或漂浮几何体。
- 开放背景本身也保留从远处深蓝、中景紫色到玩家周围红色的渐变，让空间即使没有墙面也有明确的方向、深度和危险感。
- 前进方向主要由悬浮平台、侧向抓手、钩锁锚点、光轨和远处传送门共同引导。
- 侧墙不需要连续存在，只在需要动作二时出现局部悬浮墙片、抓手框架或侧向支撑结构。
- 每个跑酷动作对应一段固定环境模块，最终按音乐顺序拼成完整流程。

### 示例流程编排

- 起始段：交替踏步，玩家熟悉左右手交替按键。
- 第一变化段：加入左墙或右墙抓手，要求玩家完成指定手 + 指定方向输入。
- 压迫段：出现低矮障碍，玩家双手下压完成滑铲。
- 高潮段：出现大空隙，玩家双手上举完成长距离跳跃。
- 过渡段：指定单手上举持续按压，触发钩锁前进。
- 收束段：混合前面几种动作，连续完成后冲向终点传送门。

### 美术方向

整体视觉是开放宇宙背景 + 悬浮跑酷轨道 + 蒸汽波纯色风 + 荧光几何结构。目标不是复杂写实，而是用低成本几何模型、纯色块、发光边缘、星空远景和颜色渐变制造强烈的节奏空间。

#### 空间氛围

- 场景位于开放悬空环境中，玩家像是在宇宙背景里的荧光跑酷航道上高速前进。
- 远景目标处采用深蓝色，象征静谧、终点和吸引力，可以结合深空、星云或传送门光晕。
- 中段逐渐过渡到紫色，作为音乐和空间推进的主色。
- 玩家周围和身后逐渐变成红色，象征危险、追逐感和失败压力。
- 整体颜色从前方深蓝，到中景紫色，再到玩家周围和身后红色，形成清晰的方向感。
- 这个颜色变化不只作用在跑酷模块上，也作用在开放宇宙背景、星云雾气、远景网格和漂浮几何体上。
- 开放背景要保留足够空旷感，避免墙面包裹玩家；只使用漂浮框架、光轨和局部几何体来强调速度。

#### 几何语言

- 地板：长条形、矩形或窄平台，表现跨越空隙。
- 侧向抓手：漂浮三角抓点、楔形凸起、短墙片段或发光斜切块。
- 滑铲障碍：低矮横梁、下压门框、平行发光条。
- 跳跃结构：远距离平台、拱形跳跃门、上扬光轨。
- 钩锁结构：上方锚点、圆环、悬挂节点、发光牵引线。
- 终点：深蓝色传送门，可以是圆形、六边形或拱门形结构。

#### 材质与 shader

- 使用简单几何体搭配纯色荧光 shader。
- 资产表面可以是低细节纯色面，边缘使用 emission 发光强调轮廓。
- 不需要复杂贴图，优先使用材质颜色、边缘光、扫描线、渐变雾和节拍闪烁。
- 成功输入时，当前模块可以闪白或增强发光。
- 失败时，玩家周围红色增强，环境可以短暂脉冲。

### 资产需求清单

- 悬浮长条地板模块。
- 左右侧悬浮三角抓点或短墙片段模块。
- 低矮滑铲障碍模块。
- 长距离跳跃平台模块。
- 钩锁锚点与牵引线模块。
- 终点传送门模块。
- 开放宇宙背景模块，可以包含星空、抽象星云、远景网格、漂浮几何体和局部侧向框架，并支持从远处蓝色到近处红色的整体环境渐变。
- 节拍提示 UI，可以是世界空间中的发光图标或轨道提示。
- 成功、失败、接近终点等状态的灯光和颜色反馈。

### Demo 范围控制

- 只做一首固定音乐或一段固定节奏片段。
- 只做一条固定路线，不做关卡编辑器。
- 只做 5 种输入动作，不扩展更多跑酷动作。
- 只做固定动画播放，不做复杂物理模拟。
- 只做环境资产和第一人称反馈，不做完整玩家模型。
- 重点验证：VR 双手输入是否有节奏感、动作是否容易理解、视觉流程是否足够酷。

## 开发任务拆分

整体开发可以分成三层：底层通用系统、单动作 Demo / 教程关卡、最终组合关卡。推荐先让节拍器和输入判定跑通，再分别制作五个动作小 Demo，最后把五个动作按音乐顺序串成完整关卡。

### 1. 通用基础系统

#### 节拍器与音乐系统

- 负责播放固定音乐或固定节奏片段。
- 根据 BPM 或手动时间轴生成 Beat Event。
- 提供统一节奏判定窗口，例如 Perfect、Good、Miss。
- 支持动态速度倍率，失败时降低音乐速度，例如从 1.0 降到 0.97。
- 音乐速度倍率需要同步影响 BPM、Beat Event 时间、输入判定窗口、动作动画节奏和关卡推进速度。
- 向动作判定脚本发送当前需要完成的动作类型、目标手、方向和持续时间。
- 输出节拍提示、成功反馈、失败反馈、关卡推进信号和失稳状态信号。

#### 输入采集系统

- 获取左右手控制器的按键状态。
- 获取左右手的位置和朝向。
- 将手部朝向简化为上、下、左、右等粗方向。
- 支持单手按键、双手同时按键、交替按键和持续按压。
- 为每个动作判定脚本提供统一输入接口。

#### 动态 BPM / 失稳系统

- 维护当前音乐速度倍率，初始值为 1.0。
- 接收所有动作判定脚本上报的成功、失败和漏键结果。
- 失败时降低速度倍率，例如 1.0 -> 0.97，并通知音乐、节拍器、动画和关卡推进系统同步变化。
- 连续成功时逐渐恢复速度倍率，例如每次小幅增加，最多恢复到 1.0。
- 恢复速度应慢于失败降速，让失败有压力，但玩家仍有把音乐救回来的空间。
- 低于 0.9 时触发音频畸变、视觉失稳和红色压迫增强。
- 低于最终失败阈值时触发被吞噬失败演出。
- 该系统应独立于具体动作脚本，避免每个动作重复处理速度和失败逻辑。

#### 关卡流程系统

- 管理当前教程段、单动作 Demo 段和完整组合关卡。
- 根据 Beat Event 推进玩家位置、镜头运动或环境运动。
- 触发动作动画、环境模块激活、成功/失败反馈和终点传送门。
- 输入失败时不立即停止动作播放，而是继续执行动作并叠加音乐降速、画面震荡和空间失稳。
- 当速度倍率低于警戒值，例如 0.9，关卡进入畸变阶段；低于最终失败阈值时触发被吞噬失败演出。
- Demo 阶段可以先做线性时间轴，不需要复杂关卡编辑器。

### 2. 背景与视觉系统

#### 开放宇宙背景

- 制作开放星空、抽象星云、远景网格和漂浮几何体。
- 背景整体保留从远处深蓝、中景紫色到玩家周围红色的空间渐变。
- 避免封闭隧道感，不使用连续侧墙或顶棚包裹玩家。
- 用光轨、漂浮框架、粒子和远处传送门引导前进方向。

#### 荧光几何材质

- 制作纯色低模材质和 emission 发光边缘。
- 支持成功时闪白或增强发光。
- 支持失败时周围红色脉冲、画面震荡或画面警告。
- 当音乐速度低于 0.9 后，背景星云、光轨、材质发光和音频可以逐渐畸变，表现世界开始失稳。
- 资产优先使用简单几何体，降低建模和贴图成本。

#### 节拍提示视觉

- 在世界空间中显示当前动作提示。
- 提示可以是发光图标、轨道标记、手势方向箭头或目标物体预亮。
- 每个动作的提示风格要统一，方便玩家从教程过渡到正式关卡。

### 3. 单动作 Demo / 教程关卡

每个动作都先制作一个独立小 Demo。它既是开发验证场景，也是最终游戏里的教程段。每个小 Demo 都包含四类交付物：判定脚本、物体资产、动作表现、反馈效果。

#### Demo A：交替踏步 / 跨越空隙

- 判定脚本：检测左右手按键是否按节拍交替输入。
- 物体资产：等距排列的悬浮长条平台和平台间空隙。
- 动作表现：成功时玩家视角或环境向下一个平台推进。
- 教程重点：让玩家理解左右手按键对应跑酷步伐。

#### Demo B：侧向抓手 / 悬空横移

- 判定脚本：检测指定手是否朝向左侧或右侧并按下按键。
- 物体资产：路线两侧的漂浮三角抓点、短墙片段或发光抓手。
- 动作表现：成功时触发侧向借力、横移或向前牵引动画。
- 教程重点：让玩家理解手部方向也参与判定。

#### Demo C：双手下压 / 滑铲

- 判定脚本：检测双手是否同时朝下并同时按键。
- 物体资产：低矮横梁、下压门框或发光障碍带。
- 动作表现：成功时镜头短暂降低或触发滑铲穿越动画。
- 教程重点：让玩家理解双手同步输入和身体姿态变化。

#### Demo D：双手上举 / 长距离跳跃

- 判定脚本：检测双手是否同时朝上并同时按键。
- 物体资产：大空隙、远距离平台、跳跃门和上扬光轨。
- 动作表现：成功时触发抛物线前进、速度提升或跳跃镜头动画。
- 教程重点：让玩家理解双手上举对应起跳和高潮动作。

#### Demo E：单手上举持续按压 / 钩锁前进

- 判定脚本：检测指定手是否朝上并持续按住按键。
- 物体资产：上方钩锁锚点、发光牵引线、环形挂点或空中轨道。
- 动作表现：成功时沿光线或弧线向前滑行，松手过早则失败。
- 教程重点：让玩家理解持续输入和长音节奏。

### 4. 最终组合关卡

- 使用同一套节拍器、输入系统和反馈系统。
- 把五个单动作 Demo 中验证过的模块组合成一条完整路线。
- 关卡结构可以是：交替踏步开场、侧向抓手变化、滑铲压迫、长跳高潮、钩锁过渡、混合动作冲向传送门。
- 音乐编排从简单到复杂，前半段像教程，后半段开始混合动作。
- 最终目标是抵达深蓝传送门，形成完整的演示闭环。

### 5. 推荐开发顺序

1. 搭建 VR 输入采集和手部方向判断。
2. 搭建节拍器、音乐播放和 Beat Event 时间轴。

4. 依次完成侧向抓手、滑铲、长跳、钩锁四个单动作 Demo。
5. 制作开放宇宙背景、蓝紫红空间渐变和基础荧光材质。
6. 将五个 Demo 整合为教程流程。
7. 制作最终组合关卡和终点传送门。
